# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Active Project: Пушок RPG

Russian-language MMO RPG companion web panel for a Discord/Telegram bot.

- **artifacts/puhok-rpg** — React + Vite + Tailwind + framer-motion frontend (17 game pages: Home, Character, Classes, Inventory, Shop, Pets, Quests, Bosses, Arena, Duel, Guilds, Auction, Crafting, Chat, Events, Premium, Admin).
- **artifacts/api-server** — Express 5 backend with cookie-based session (`puhok_sid`). In-memory store seeds 8 hero classes, raid bosses, arena/duel opponents, guilds, auction lots, recipes, chat (with bot replies to "Пушок"), events, servers, premium plans, and admin economy.
- **lib/api-spec** — OpenAPI source of truth.
- **lib/api-zod / lib/api-client-react** — generated by orval from the OpenAPI spec. The Zod index re-exports only `./generated/api`; orval is configured with `indexFiles: false`.

After editing the OpenAPI spec, run `pnpm --filter @workspace/api-spec run codegen`. After editing routes/store, restart the api-server workflow.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.
